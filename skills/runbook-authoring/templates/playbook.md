# Playbook Template

A playbook is branching: choose a path based on observed evidence.

## Situation

Describe the symptom, affected users/services, and current impact.

## Entry criteria

When this playbook should be used and when it should not.

## Decision tree

| Observation | Interpretation | Next action | Risk |
|---|---|---|---|
|  |  |  |  |

## Commands

| Command | Purpose | Expected normal signal | Abnormal signal |
|---|---|---|---|
|  |  |  |  |

## Mitigation options

Options, approval requirements, expected effects, and rollback paths.

## Approval gates

Actions that must stop until explicit approval is received.

## Communication

Who to update, when, and with what status format.

## Closure criteria

Conditions required to declare the issue stable or resolved.
